    # Koa2重构后端

    • 比较

      • 不使用框架：通过nodejs底层搭建消息服务器，基于底层http-server，request，response来做各种解析，从cookie到session，甚至body集都要自行处理和维护
      • Express：大而全，有很多优秀的中间层组件可以直接用；基于ES5语法，异步需要回调，回调层次太多，层层嵌套
      • Koa2：小而精；更优雅地处理异步流程（用同步的方式），避免了回调，在流程的控制层面更加机动和灵活

    • 学习Koa2

      • 问题

        • 405错误，method not allowed

          默认提交的header参数"content-type"为“application/json”，而传入的是form-rulencoded格式，要明确参数

        • 全局拦截不跳转至登录页面

          最外层if没有写else结构，逻辑不完整造成了不跳转

        • 表单点击提交需要两次

          form中的submit带有默认的提交和跳转动作，在点击submit时，执行的时默认的提交和跳转

      • 中间件

        • koa-body

          对文件上传和POST JSON，设置json打大小和上传文件的大小, 都比较友好

        • koa-session

        • bcrypt

          不可逆加密算法，无法通过解密密文得到明文

      • 保持登录状态

        • 用koa-session在cookie中设置sessionId，前端路由设置全局拦截，如果存在session则通过，否则跳转到登录页面

          const koa_session = require('koa-session')
const session_signed_key = ["some secret hurr"]
const seesion_config = {
    key: 'UserName',
    path: '/',
    maxAge: 60*60*24*7,
    autoCommit: true,
    overwrite:true,
    httpOnly:false,
    signed: true,
    rolling:true,
    renew:false,
    credentials: true
}
const session = koa_session(seesion_config, app)
app.keys = session_signed_key;
app.use(session)

          //全局拦截
router.beforeEach((to, from, next) => {
  if (to.matched.some(record => record.meta.requireAuth)) {
    console.log(getCookie('UserName'))
    if (getCookie('UserName') != '') {
      next()
    } else {
      console.log('to login')
      next({
        path: '/login'
      })
    }
  } else {
    next()
  }
})

    • 自动化部署

      每次更新本地代码之后都要手动上传服务器并且重启node服务，非常繁琐，可以通过pm2配置进行自动化部署，本地更新代码之后推送到github，并实现服务器自动更新部署并自动重启服务

      • 本地和服务器git环境配置好，都有权限拉取项目代码，可以添加本地ssh到服务器，这样可以节省每次发布部署代码都要频繁输入服务器密码的麻烦

        ssh-copy-id -i ./ssh/id_rsa.pub root@xxxx
