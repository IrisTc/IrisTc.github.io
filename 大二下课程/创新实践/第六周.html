# 第六周

  • 怎么解决在云函数上实现自动实时编译

  • 在入口函数写脚本即可执行generate那个脚本

    • 当运行 nuxt generate 命令或在编码中调用 nuxt.generate() 时，Nuxt.js 会使用 generate 属性的配置。
    • 用serverless创建的框架和本地手脚架创建的一样
    • 配置里面是把npm run build打包编译的.nuxt文件夹传上云？
    • 无法生成新的文件，整个项目已经被打包编译成了一个文件

  • 怎么将打包的dist文件夹传到cos

    • 用js写上传文件的函数
    • 云函数由向 腾讯云对象存储 COS (opens new window) 上传对象文件操作、向 腾讯云消息队列 CMQ (opens new window) 中投递消息等事件触发

# 问题

  • 云函数出现传不上去的情况

  • 直接通过serverless的component上传是直接压缩打包的，找不到入口文件在哪

  • nuxt.build is not a function

    • 研究了一下nuxt的源码

    • const { Nuxt, Builder, Generator } = require('nuxt')

// Import and Set Nuxt.js options
const config = require('./nuxt.config.js')
config.dev = process.env.NODE_ENV === 'production'

async function start() {
    // Init Nuxt.js
    const nuxt = new Nuxt(config)

    await nuxt.ready()
    // Build only in dev mode
    const builder = new Builder(nuxt)
    const generator =  new Generator(nuxt, builder)
    await generator.generate()
}
