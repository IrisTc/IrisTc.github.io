    # 学习日志小程序立项报告

    18271407 陶莎

    # 项目概述

    # 项目背景
    • 针对低效学习的问题。无论考研式的学习还是想要提高技术能力的学习，都普遍存在学习效率低下的问题，花费大量时间却是在低水平勤奋，想找到解决这种低效学习的症结的原因所在，就必须有意识地在学习之后思考和寻找更高效的学习方式，工欲善其事必先利其器。
    • 今天学习了哪些东西，今天进步了多少？（如何给自己一个客观如实的答案？）
      • 这个如何测量，只知道今天学习的大体内容，如果具体、详尽的回想今天所学的内容，会发现细思极恐，因为回顾出来的只有一个模模糊糊的概念，学习的任何细节都不清晰了
      • 最好的方法是记录下来，把一天所学变成一个客观东西，这样就很容易测量，记录了多少就是进步了多少，否则一天下来就是糊里糊涂的，脑袋完全混乱不堪，感觉一天下来什么都没学到，完全在浪费时间，这是最麻烦也是最可怕的一种状态。
      • 利用同样的方法，今天记录了哪些知识点，都有哪些是之前不会的或者完全没想到的，把他们纠正过来，把不会的弄懂也是进步，这样记录多少就有多少进步，记录下的就是进步的内容
      • 经过上面的步骤，回顾一下新掌握的知识，将所学的用起来试一试，在使用过程中就会发现有很多漏洞，有些是没有考虑到的细节，赶紧补上，这个发现漏洞、未知的、不懂的、遗忘的、答不上来不上去，查漏补缺发现新的进步空间
    • 当前主流打卡移动端App虽然功能齐全，但是使用起来较为繁琐，必须得下载注册使用

    综上所述，学习日志的目的就是为广大学习者提供一个可以记录自己的学习情况的小程序。

    # 项目概况

    • 功能说明

      • 每条记录包括学习内容，时间范围，标签

      • 用户可对自己的学习记录进行增、删、改、查（通过日期查看一天的记录）

      • 用户可以根据自己所学内容的特殊场景自定义标签，比如学习前端技术时可以增加做项目、查资料、写博客、框架学习等标签，满足需求为定制需求与简易化操作需求

      • 用户可以查看一段时间内学习情况的统计结果（通过标签进行统计）

      • 邀请好友开启激励模式，彼此可以看见对方的学习日志
