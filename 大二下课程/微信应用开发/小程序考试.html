# 小程序考试

  • 为什么小程序不需要语义标记

  • 小程序引入了自己类似HTML标签的组件， 按功能划分，面向交互

  • HTML标记是按语义划分，面向内容

  • 从功能上讲，text或richtext两个组件有什么不同？

    • 文本和富文本
    • 文本组件中只能插入文字
    • 富文本解析HTML内容或者节点列表，在前端渲染出来

  • 小程序如何实现页面搜索功能？ ,[参考](https://developers.weixin.qq.com/miniprogram/dev/framework/search/seo.html#1. 小程序里跳转的页面 (url)%20%E5%8F%AF%E8%A2%AB%E7%9B%B4%E6%8E%A5%E6%89%93%E5%BC%80%E3%80%82)

    • 输入框，提交按钮
    • js处理，后台返回数据，更新前端绑定数据列表

  • 如何理解页面模板与页面实例？请举一实例说明

  • 你用过哪些WEB前端模板引擎或WEB后端模板引擎？和wxml模板相比，有什么区别和共同之处？wxml模板支持嵌入函数或方法吗？

    • template-web。都用{{}}符号进行模板替换，template-web需要引入模板引擎js，需要制作模板，需要写相关语法；而wxml不需要自己引入，直接在标签中添加相关语法即可
    • 支持事件绑定函数和方法

  • 从上例中，小程序是否支持ES5、ES6等语法及标准库？请举一实例说明

  • 支持，如果需要支持ES6语法则需要在设置中配置ES6转ES5，小程序中可以封装promise对象

  • 请列举出a标记与navigator组件在使用上的不同之处

  • a标记是在src属性里面写入链接，navigator在url属性里面写入链接，还可以选择open-type的属性

  • open-type="navigate"等价于API的　wx.navigateTo　而wx.navigateTo的url是需要跳转的应用内非 tabBar 的页面的路径

    open-type="redirect"等价于API的　wx.redirectTo　而wx.redirectTo的url是需要跳转的应用内非 tabBar 的页面的路径

    open-type="switchTab"等价于API的　wx.switchTab而wx.switchTab的url是需要跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面

  • 传统Web页面编程中，支持Page对象吗？ Page对象对Web前端编程带来了哪些好处？

  • 不支持

  • 为应用程序的每个页面创建一个Page对象，具有表示给定页面上可用服务的方法，可以在这些对象中封装系统的所有实现细节（即HTML元素，等待DOM更新等）

  • require('./data.json')这行语句能正常工作吗？如果不能，为什么？可否有方法使其能正常工作？

  • 不能。 json是数据文件，需要执行文件读取与解析等操作

  • 可以写一个http-server启动服务访问json文件

  • 本地文件服务和数据库服务采用的何种通讯协议和数据交换格式?

    • 文件协议，文本格式？

    • http协议，json数据交换格式

  • 请结合本例，说明JSON.stringify()和JSON.parse()两个函数的功能？

    • JSON.stringify()用于将JavaScript值转换为JSON字符串
    • JSON.parse()将字符串转换为JavaScript值

  • 本例中，let page=this 背后原理是什么？

  • page()函数用来注册一个页面，即获取当前页面的page对象实例

  • wx.request与ajax有何区别与联系？

    • 用法类似
    • wx.request请求的url必须是https协议，ajax不限制协议，但不能跨域请求

  • 小程序表单控件与HTML相比，有哪些异同？

    • 都用form标签标识表单，小程序组件是对html标记的一种包装，提供更多的属性选择，提供统一或更简易的使用方式

  • 小程序引擎将表单封装到event对象，与传统Web表单的DOM取值方式相比，有什么优势?

  • 小程序端不需要通过DOM找到节点，直接在form上绑定事件后在js中通过e.detail.value获取

  • 什么是运行时(run time)？什么是动态编程(Dynamic Programming)?什么是动态编程语言？

  • 运行时是指将数据类型的确定由编译时推迟到了运行时

  • 动态编程：在运行时可以改变其结构的语言 :例如新的函数、对象、甚至代码可以被引进,已有的函数可以被删除或是其他结构上的变化

  • JavaScript、Python、Ruby等，一般不需要进行编译即可运行的语言

  • 命令式编程与反应式编程的区别？

    • 命令式编程是以命令为主的，给机器提供一条一条的命令序列让其原封不动的执行。而反应式编程用多种方式使用数据流，合并，过滤和转换等，抽象层次更高，关注于如何把小的方法连接起来，而不是方法内部如何实现的

  • 请举出实例，说明什么进程？什么是线程？两者的区别与联系？

    • 进程是CPU资源分配的最小单位，Chrome浏览器就是多进程的，没打开一个tab标签页，就相当于创建了一个独立的浏览器进程，各个标签页之间互不影响
    • 每个进程中有很多线程，比如GUI渲染线程，负责渲染浏览器界面，解析HTML，css，构建DOM树等等，JS引擎线程，也成JS内核，负责处理JavaScript脚本程序还有定时触发器线程

  • 请举出实例，说明什么是跨线程调用？跨进程调用？

  • 在Web开发中，还存在哪些跨页面(或跨线程、跨进程)传递信息的方法？请找出在小程序中的对应方法

  • input hidden 与 wx:if这两种隐藏的方式有何不同?

    • input hidden 会在页面渲染出来，只是隐藏之后不会呈现
    • wx:if在隐藏时不会渲染
