# 小程序学习记录

  • 记录框的高度总是显示为手机屏幕高度的70%,可以获取用户系统信息

    wx.getSystemInfo({
      success: function (res) {
        // 获取可使用窗口宽度
        let clientHeight = res.windowHeight;
        // 获取可使用窗口高度
        let clientWidth = res.windowWidth;
        // 算出比例
        let ratio = 750 / clientWidth;
        // 算出高度(单位rpx)
        let height = clientHeight * ratio;
        console.log(height);
        that.setData({
          height: (height * 0.7) + 'rpx'
        })
      }
    })

  • 小程序云数据库在小程序段一次查询最多返回20条数据，云函数最多返回100条数据

    • 使用云函数查询dailyLog中一天的数据实时渲染

      // 云函数入口文件
const cloud = require('wx-server-sdk')

//初始化环境
cloud.init()
const db = cloud.database()

// 云函数入口函数
exports.main = async (event, context) => {
    return await db.collection('dailyLog').where({
      _openid: event.openid
    }).get()
}

    • 云函数的调用

      wx.cloud.callFunction({
  name: 'getlogs',
  data: {
    openid: app.globalData.openid
  },
  success: res => {
    that.setData({
      logs: res.data
    })
    console.log('云函数getlogs:', res)
  },
  fail: err => {
    console.error('[云函数] [getlogs] 调用失败', err)
  }
})

  • 获取用户信息（openid）用于查询数据库

    wx.cloud.callFunction({
  name: 'login',
  data: {},
  success: res => {
    console.log('[云函数] [login] user openid: ', res.result.openid)
    app.globalData.openid = res.result.openid
  },
  fail: err => {
    console.error('[云函数] [login] 调用失败', err)
  }
})

    操作成功了，理应回调success，却没有；而在小程序端，一样的代码，却能成功回调

    • 找资料有人表示 在 wx-server-sdk 中不再兼容 success、fail、complete 回调，总是只会返回 Promise

  • 生命周期问题

    • 调用云函数login获取openid之后，再调用数据库查询函数时总是显示openid未定义，而且显示时间在openid获取到了之后

    • 小程序的生命周期

      • 整个小程序只有一个 App 实例，是全部页面共享的 ， 可以通过 getApp 方法获取到全局唯一的 App 实例，获取App上的数据或调用开发者注册在 App 上的函数 。 App() 必须在 app.js 中调用，必须调用且只能调用一次

      ​ app()的参数：

      • 小程序启动之后，在 app.js 定义的 App 实例的 onLaunch 回调会被执行

    • 页面的生命周期

      • 页面生命最气函数就是当你每进入/切换到一个新的页面的时候，就会调用的生命周期函数。Page(object)函数用来注册一个页面，接受一个Object类型参数， 其指定页面的初始数据、生命周期回调、事件处理函数等

      • 在渲染完界面之后，页面实例就会收到一个 onLoad 的回调，你可以在这个回调处理你的逻辑

    • 因为onLunch()是小程序初始化完成时触发，全局只触发一次，可以在这里获取openid放入到globalData中，页面在onLoad中调用获取数据库的云函数即可

  • WeUI的使用

    • 写“我的”页面的时候感觉都是一个个的组件，自己写样式有点麻烦，想到了WeUI

    • 这是一套基于样式库weui-wxss (opens new window)开发的小程序扩展组件库，同微信原生视觉体验一致的UI组件库，由微信官方设计团队和小程序团队为微信小程序量身设计，令用户的使用感知更加统一。

    • 直接将github上的项目copy下来，把dist文件夹导入微信开发者工具，可以更加方便的找到相对应的样式组件

    • style文件夹复制到自己的小程序里面，在app.wxss中全局引用即可

      @import '/style/weui.wxss';

  • POST请求的坑！

    • POST 请求会将data的值放在Request Payload里面，而不是Query String Parameters里面，后端服务器如果不注意，就无法取到数据，这个导致我页面重新获取数据时每次都是一样的。
