# 基于开发者角度的小程序调研

  • 小程序的代码结构与布局

    pages 用于指定小程序由哪些页面组成，每一项都对应一个页面的路径（含文件名） 信息。文件名不需要写文件后缀，框架会自动去寻找对应位置的 .json, .js, .wxml, .wxss 四个文件进行处理，每个page的目录如下

    • page

      • 骨架文件wxml

      • 样式文件wxss

      • 业务文件js

      • 页面配置json(非必须)

        作为页面的个性化配置，只能定义app.json中window相关的配置项

    • 全局文件

      • 全局配置app.json

        小程序当前项目的程序配置，每个小程序是由多个页面组成，但在这些页面之上，存在所有页面共用的内容，可以配置小程序的页面路径，窗口表现（比如标题颜色），底部tab，网络请求超时，调试模式等

      • 全局样式app.wxss

        所有页面都能覆盖的样式

      • 全局业务app.js

        整个小程序第一个入口文件，有app函数，是小程序内置的，每个项目必须有且只有一个，必须出现在app.js中，这个函数中描述这个app的一些生命周期和全局数据

    • 组件component

      小程序本身提供丰富的组件，也支持自定义组件，可以将页面相同的功能模块抽象成自定义组件，以便在不同的页面中复用，包括以下文件

      • wxml、wxss、js、json

    • 自动生成的环境文件project.config.json

      为了减少开发者重复配置开发工具，通过json数据格式，每个小程序项目都在配置文件里定义了开发者习惯的配置参数，无论开发环境如何变更，只要载入同一个项目的代码包，开发者工具就通过project.config.json自动回复开发项目的个性化配置，其中包括编辑器的颜色、编辑设置、代码上传自动压缩等一系列选项

  • 小程序和H5的开发技术

    • 开发语言

      微信开发了一套wxml标签语言和wxss样式语言，相当于标准的html和css，写法和形式都是和H5应用的语言一样的，只是有些标签是微信自己定义的，逻辑层还是传统的js

    • 开发框架

      写H5前端时有vue框架，现在H5应用主流的框架还有React、Angular等，微信开发了自己的小程序框架，也是将逻辑层和视图层分离，因为腾讯团队是借鉴了vue的语法风格和功能特性，支持了vue的诸多特征，比如父子组件、组件之间的通信、computed计算属性、props传值、slot槽分发等等

  • 小程序和H5的开发流程

    微信的开发流程更简洁

    • H5项目用开发工具加浏览器Device Mode预览的模式，而小程序基于自己的开发者工具，可以实现同步本地文件，开发调试，编译，预览等，web网站需要部署到服务器上，而小程序替开发者完成了这一步，直接上传云服务器便可上线，但是小程序需要审核，而web端不需要

    • 数据绑定

      H5应用为了便于数据传递，需要借助框架或者模板引擎进行数据绑定， 小程序借鉴了很多流行框架都有的数据绑定思想；同时出于性能考虑，只实现了单向数据绑定，即只支持从逻辑层传递到 UI 层的数据绑定 ，page对象原型链上的setData 方法接受一个对象，以 key-value 的形式将 this.data 中的 key 对应的值设置为 value。setData 执行后会通知逻辑层执行重新渲染，并立刻渲染 UI，显示更新的数据。

    • 系统权限

    微信小程序相对于 H5 能获得更多的系统权限，比如：网络通信状态、数据缓存能力等，这些系统级权限都可以和微信小程序无缝衔接，H5虽然有摄像头之类的权限，但是重度依赖浏览器能力，兼容性有限

    • api

      小程序开发框架提供丰富的微信原生 API，API是应用程序接口，是一些预先定义的函数，让开发人员无需访问源码就可以访问一组例程的能力，可以方便的调起微信提供的能力，如获取用户信息，本地存储，支付功能等 ，比H5应用需要自己编写接口方便很多

  • 小程序和H5的开发环境

    • 浏览器兼容问题

      H5应用需要考虑浏览器低版本不支持的问题，而小程序开发工具时一种基于系统层的框架，它的组件、UI都是确定好了的，不用考虑兼容问题，虽然在iOS和Android客户端的JavaScript语法和API支持不一样， 开发者可以通过开启 ES6 转 ES5 的功能来规避，小程序基础库内置了必要的Polyfill，来弥补API的差异

    • 开发工具

      H5的开发工具的选择多样，有vscode、HBuilder、WebStorm等等，根据开发者的习惯和喜好选择，而微信小程序提供了统一的开发者工具，并且集项目创建、开发、调试于一身，脚手架起步，预览自动打包，可以实时调试更新，控制台有网络请求和console，集成了Chrome浏览器的功能

    • 调试

      H5在PC端调试直接打开不同的浏览器即可，而真机调试需要将手机和电脑连接或者下载第三方应用模拟真机调试，而小程序的模拟器模拟微信小程序在客户端真实的逻辑表现，对于绝大部分的 API 均能够在模拟器上呈现出正确的状态， 可以实时调试，预览自动打包，小程序上也可以发起一个真机远程调试流程，点击开发者工具的工具栏上 “真机调试” 按钮，工具就会将本地代码进行处理打包并上传，然后使用手机客户端扫描二维码即可弹出调试窗口， 使用手机扫描此二维码，即可开始远程调试 。在真机上，如果想要查看 console API 输出的日志内容和额外的调试信息，可以在点击屏幕右上角的按钮打开的菜单里选择「打开调试」即可

  • 语言区别

    • WXML(WeXin Markuo Language)和HTML类似，也有标签和属性，但针对小程序平台做了优化，相较于HTML，小程序的标签显得更加简洁，比如div、section、header等块级标签统一为了view、p、span、b等，文字类标签统一为text，同时新增很多实用标签，比如picker滚动选择器、map地产图、web-view网页容器等等

    • WXSS(WeXin Style Sheets)是微信定义的一套样式语言，其具有CSS大部分特性，同时为了更适合微信小程序，WXSS对CSS进行了扩充以及修改。小程序使用rpx作为尺寸单位，屏幕宽度固定位750rpx，设置了rpx单位的元素可以根据屏幕宽度进行自适应，另外，微信小程序也支持rem尺寸单位。小程序没有html、body标签，如果想要设置页面的样式，可以直接使用page选择器

      page{ background: #fff;}

    • 小程序中的JavaScript没有window、document等变量，大部分浏览器中全局方法会被禁用，比如alert，也有部分支持，比如setTimeout、encodeURIComponent等， 因为没有DOMapi，所以前端常用的库比如 jQuery在小程序中无法运行，但是开发者工具1.02以上开始小程序支持使用npm安装第三方库

  • 小程序的专属配置

    • 页面窗口表现配置

      一些针对微信统一界面的配置，比如标题颜色，tab栏等

    • 页面自定义组件配置

    "usingComponents": { "head": "/components/head" }

  • 编译

    小程序的编译即合并编译，将所有页面的js合并到一个文件appservice.js中，可以加快加载速度

  • 预处理技术

    • scss预处理器可以将sass文件转换成css，其基于Ruby， 在服务器端处理完成 ，所以必须安装ruby
    • Babel 是一个工具链，主要用于将 ECMAScript 2015+ 版本的代码转换为向后兼容的JavaScript 语法，以便能够运行在当前和旧版本的浏览器或其他环境中，可以实现语法转换，通过 Polyfill 方式在目标环境中添加缺失的特性 (通过 @babel/polyfill (opens new window) 模块)，源码转换(codemods)
    • 小程序提供了几个默认的预处理，ES6转ES5，上传代码时样式自动补全、自动压缩

  • 小程序开发的优势

    • 框架

      前后端分离，视图层和逻辑层分离，数据绑定高效

      传统的HTML需要通过JavaScript对DOM结点进行操作，才能动态改变页面内容，或者自己使用第三方模板引擎。而在WXML里，数据可以跟页面结构进行绑定，JavaScript只对数据进行操作，而这些数据的修改，最终会反馈给绑定数据的页面结构并动态更新，这种机制类似于Vue.js的声明式渲染，实现了页面和数据的分离，对于开发来说无疑更加高效

    • 组件

      小程序提供丰富的组件，并且支持自定义组件

      小程序提供了视图容器、表单组件、媒体组件、画布组件等一系列基础组件，有些模块可能会在多个页面中用到，逻辑也相对独立，比如相同样式的banner，用传统的开发方法实现是很繁琐的，小程序提供了让开发者自己创建组件的特性，这会极大提高代码复用率，还有很强的代码分离性，开发者可以减少大量重复的编写

    • 工具

      官方的微信开发者工具，集项目创建、开发、调试于一身，脚手架起步，预览自动打包，可以实时调试更新，控制台有网络请求和console，集成了Chrome浏览器的功能，使开发更加高效

    • 语言

      相较于HTML，小程序的标签显得更加简洁，同时增加了很多实用标签，比如picker滚动选择器、map地图等等

  • 小程序技术的职业前景

    • 与前端分离

      小程序为企业提供了流量入口，企业可以通过小程序推广自己的产品，未来极有可能出现大量专有的小程序开发工程师职位，类似于iOS和Android工程师，现在在招聘网站上已经出现微信小程序程序员的招聘信息

    • 必备技能

      对于前端开发而言，正是因为微信小程序其开发成本低，用户流量巨大，开发迭代迅速等特点，各个企业都在定制自己的小程序，小程序开发也就成了前端开发工程师必会的一个技能

    • 发展前景

      小程序现在在外包市场出现的频次非常之高，反映了大众的需求，潜在的生态圈很大，每个人都用过小程序，小程序和微信粘度高，自然而然有了庞大了用户体系，不断发展，进一步提升自己的开放能力，借助这种开放能力开发者能够在小程序上做更多的事情
