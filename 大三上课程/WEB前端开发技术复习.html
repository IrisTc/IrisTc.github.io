    # WEB前端开发技术复习

    # 基础

    1、用户场景： 用户使用Web浏览器访问某购物网站，在地址栏里输入：http://x.com/index.php ，然后按回车键，看到了琳琅满目的商品首页

    2、用户场景： 用户通过浏览器访问某网站的用户注册页面。如：http://x.com/users/new.php

    （1）当用户未填写用户名，提交后，浏览器立即弹出对话框，提示用户名为必填

    （2）当用户填写了一个常见的用户名如：tom，提交后，浏览器弹出对话框，提示该用户名已被占用

    （3）当用户终于按要求正确填写所有字段，提交后，页面跳转至网站首页

    请基于上述用户场景，从Web浏览器和Web服务器、HTTP请求和响应、客户端检验与服务端校验、客户端渲染和服务端渲染等多个技术角度，说明在浏览器背后，经历了哪些技术阶段和发生了什么？

    • 获得本机IP和本地DNS服务器IP
    • DNS请求，解析目的URL的IP地址
      • 本地DNS服务器接收到DNS请求，查询缓存，如果有该地址则返回，否则请求根服务器
    • 进行HTTP请求，建立TCP连接，请求Web页面信息
      • 发送HTTP报文
      • 建立TCP连接
      • 三次握手
      • 服务器返回Response，即WEB页面对象
    • 浏览器加载并解析WEB对象显示
    • 客户端校验：本机的浏览器上通过JavaScript脚本代码对用户的输入信息进行一定规则的校验，优点是快捷，用户体验好，减少服务器端的压力，但不可靠，可绕过
    • 服务端校验：一般用户提交表单后会进行服务端校验信息，如重名问题，后端服务会连接数据库进行查询
    • 客户端渲染：前端获取数据后生成DOM树，SPA应用用户点击时不会形成频繁的跳转，但首屏响应速度慢，不利于SEO优化
    • 服务端渲染：DOM树在服务端生成，直接返回完整的HTML给前端，不利于前后端分离，加大服务器的压力

    2、请列举除浮动之外的至少三种列式布局技术，并一一说明其各自的适用场景、技术优势和不足之处

    # 列式布局技术

    • 浮动
      • 这是一种比较便利的实现方式，无需额外的元素辅助定位，同时兼容性也比较优秀。但有一个缺点就是该布局方式只能实现左右两列宽度固定，中间自适应这一种三列布局，灵活性不强。
    • 绝对定位
      • 如果中间栏含有最小宽度限制，或是含有宽度的内部元素，当浏览器宽度小到一定程度，会发生层重叠的情况
    • Flex布局
      • Flexbox布局常用于设计比较复杂的页面，可以轻松的实现屏幕和浏览器窗口大小发生变化时保持元素的相对位置和大小不变，同时减少了依赖于浮动布局实现元素位置的定义以及重置元素的大小
      • 浏览器兼容性比较差，只能兼容到ie9及以上
    • Grid布局
      • 可以更灵活构建和控制自定义网格 ; 能够将网页分成具有简单属性的行和列来完成我们需要的网格布局
      • 兼容性不好

    # 名词解释

    • DOM

      文档对象模型，document object model，是W3C的标准，所有浏览器公共遵循的标准。DOM从更高的维度观察 html文档，将其视为一个树型结构的节点对象集合，并提供一组标准化的操作接口，从而简化了编程，提高了开发效率。

    • BOM

      浏览器对象模型，browser object model，BOM是各个浏览器厂商根据DOM在各自浏览器上的实现，主要处理浏览器窗口和框架，是为了控制浏览器的行为而出现的接口。 通过BOM, 网页开发者可以了解到浏览器的信息和运行时的信息，还可以通过程序化的方式操控浏览器

    • CSS伪类

      CSS伪类是W3C里制定的一套选择器的特殊状态，用来添加一些选择器的特殊效果，例如:focus、:active等

    • CSS伪元素

      伪元素是DOM树没有定义的虚拟元素，不宜元素为最小选择单元，它选择的是元素指定内容，比如::before表示选择元素内容之前的内容

    • jQuery

      jQuery是一个JavaScript库，它封装了JavaScript常用的功能代码，提供一种简便的JavaScript设计模式，优化HTML文档操作、时间处理、动画设计、Ajax交互，极大地简化了JavaScript编程。它具有独特的链式语法和短小清晰的多功能接口，具有高效灵活的css选择器，并且可拓展，拥有边界的插件拓展机制和丰富的插件

    • SCSS

      SCSS是一种CSS预处理语言，是CSS拓展语言，它在css语法的基础上，允许使用变量、嵌套规则、混合、导入、继承等功能，使css更加强大和优雅

    • 响应式设计

      页面的设计与开发根据用户行为以及设备环境（系统平台、屏幕尺寸、屏幕定向等）进行相应的响应和调整，包括弹性网格和布局、图片、CSS media query的使用等，即页面有能力去自动相应用户的设备环境，响应式网页设计就是一个网站能够兼容多个终端，而不是为每个终端做一个特定的版本

    • 媒体查询

      使用媒体查询可以针对不同的媒体类型定义不同的样式。@media可以针对不同的屏幕尺寸设置不同的样式，特别是在设计响应式页面的时候。当你重置浏览器大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面。

    • cookie

      Cookie用于存储web页面的用户信息。Cookie是一些数据，存储在你电脑上的文本文件中，以键值对的形式存储，当浏览器从服务器上请求web页面时，属于该页面的cookie也会被添加到该请求中，服务端可以通过这种方式获取用户信息

    • Nodejs

      Nodejs就是运行在服务端的JavaScript，是一个事件驱动I/O服务端JavaScript环境，基于Google的V8引擎，V8引擎执行JavaScript速度非常快，性能非常好。

    • 跨域访问

      浏览器从一个域名的网页去请求另一个域名的资源时，只要域名、端口、协议有一个不同，即跨域访问

    • 同源策略

      同源策略是一种约定，是浏览器基本安全功能。同源策略是浏览器行为，为了保护本地数据不被JavaScript代码获取回来的数据污染，因此拦截客户端发出的请求回来的数据接收，即不同源的客户端脚本在没有明确授权的情况下，不能读写对方资源。

    • 事件驱动

      事件是可以被控件识别的操作，应用程序需要对这种事情做出相应。事件驱动是指程序的执行由事件决定，所以事件驱动思维是指从事件响应触发来完成应用的设计与编程。

    • 定时器

      setTimeout(code,time)指定时间后执行；setInterval(code,time)指定周期执行

      js运行环境提供的一种执行代码的机制。通过定时器，开发者可以不依赖于用户的操作而自动调用执行某些代码，且支持异步执行。

    • URL

      Uniform Resource Locator，统一资源定位器，用于为WEB网上的每一个资源或页面赋予一个唯一的地址。

    # 辨析题

    • 安全字体、网络字体、图标字体
      • 在 Windows,Mac,linux 等各类操作系统上均可使用的字体资源，称之安全字体
      • 网络字体即在线字体，可以通过Web服务的方式进行访问
      • 图标字体是使用微小图像而不是字母形式的字体。 与字符一样，每个图标字体都是矢量元素，可根据需要进行伸缩，并可以使用CSS样式进行修改。
    • Javascript与node.js
      • JavaScript本身不提供任何与输入输出相关的API，都要靠宿主环境提供，前端嵌入的宿主环境是浏览器，后端是嵌入服务器环境
      • 前端JavaScript是基于浏览器的JavaScript，调用JS Web API，如DOM、BOM等
      • 后端Node.js是基于服务端的JavaScript，调用Node.js API，处理http、文件等
    • DOM、HTML DOM 与 SVG DOM
      • DOM分为三部分：核心DOM，HTML DOM和XML DOM
      • 核心DOM包括遍历DOM树、对接点的操作，而HTML DOM是以一种简便的方法访问DOM树
      • SVG DOM将以Adobe SVG Viewer提供的属性和方法为准，因为不同解析器对JavaScript以及相关的属性和方法支持的程度不同，有些方法和属性是某个解析器所特有的。SVG支持DOM2标准。
    • 窗口、文档
      • window对象是浏览器中打开的窗口，而document对象是浏览器窗口中的HTML文档
      • window是全局对象，根对象，而document也是此对象的一个属性
    • 客户端本地存储、服务端网络存储
      • 客户端本地存储即将数据存储在本地的方法，可以使用cookie或者浏览器支持的内置本地缓存技术，比如localStorage和sessionStorage，一般只用来存储少量数据
      • 服务端网络存储，即服务端保存数据，可以保存用户的持久数据，一般使用数据库或者云存储
    • 同步调用、异步调用
      • 同步调用就是客户端等待调用执行完成并返回结果，是一种阻塞式调用方式，即客户端一直阻塞等待直到服务器返回结果
      • 异步调用就是客户端不等待调用执行返回结果，是一种非阻塞式调用方法。
    • URL绝对地址、URL相对地址
      • 绝对地址是指文件或目录在硬盘上真正的路径，即相对根目录的地址
      • 相对地址即相对于当前页的地址
    • 请指出C:\www\index.html 、 file:///C:/www/index.html 和 http://x.com/index.html 这三者的区别？以及各自的功能限制？
      • 第一个是index.html的绝对路径
      • 第二个是本地浏览器中打开index.html文件的路径
      • 第三个是某个网页URL，可以通过网络访问
    • 静态定位、浮动定位、绝对定位、固定定位
      • 静态定位：按照标准文档流的规则来放置元素
      • 浮动定位：元素脱离标准文档流，元素停靠在浮层元素的左边或右边，或者停靠在已经浮动的元素的左边或右边
      • 绝对定位：通过设置定位的父级元素或祖先元素来进行定位，脱离标准文档流
      • 固定定位：相对于浏览器可视窗口进行定位，不留页面的滚动而改变
    • 前端模板引擎、后端模板引擎
      • WEB模板引擎负责组装数据到HTML页面中
      • 前端模板引擎：工作在浏览器中，比如art-template
      • 后端模板引擎：工作在服务器端，比如PHP、JSP等
    • 设计时、构建时(build)、编译时、运行时
      • 设计时指代码开发的所有过程
      • 构建时就是将所有源代码编译成目标文件，再链接成可执行文件
      • 编译时就是正在编译的时候，编译即边以及将源代码翻译成机器能识别的代码
      • 运行时就是代码被装在到内存中，可被执行
    • localStorage 、 sessionStorage 与cookie
      • cookie的大小限制在4KB左右，一般有服务器生成，可设置失效时间，cookie每次回携带在http头中
      • 其他两个的大小一般为5MB，仅在浏览器保存，localStorage除非被清除，否则永久保存，sessionStorage仅在当前会话下有效，关闭页面或浏览器后被清除
    • http协议、https协议
      • HTTP协议传输的数据都是未加密的，连接是无状态的，端口为80
      • HTTPS协议是具有安全性的SSL加密传输协议，是可进行加密传输、身份认证的网络协议，HTTPS通过HTTP进行通信，但利用SSL/TLS来加密数据包。比HTTP协议更加安全，端口为443
