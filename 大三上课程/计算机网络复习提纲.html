    # 第二章 计算机网络协议与体系结构

    • 三个要素
      • 语法、语义、同步
    • OSI参考模型
      • 七层：应用层、表示层、会话层、运输层、网络层、数据链路层、物理层
      • 五层：应用层、运输层、网络层、数据链路层、物理层
    • TCP/IP模型
      • 应用层
        • HTTP、SMTP、FTP、Telnet、SNMP、DNS等
      • TCP层——运输层
        • 端到端的连接和可靠通信，只存在于端节点中
        • 2个协议：传输控制协议（TCP）、用户数据报协议（UDP）
          • TCP：提供可靠数据传输、提供面向连接的服务，提供按字节流的有序、可靠传输，可以实现连接管理、差错控制、流量控制、拥塞控制等
          • UCP：提供无连接的服务，用于不需要或无法实现面向连接的网络应用中
      • IP层——网络层
        • 五种协议：IP、ICMP、IGMP、ARP、RARP
        • 不可靠、无连接的数据包协议，“尽力而为”
      • 网络接口层——数据链路层、物理层
        • 数据链路层协议：网络适配器中（网卡）
        • 应用层、TCP层、IP层：操作系统中
    • 两种模型比较
      • 重要区别：可靠性问题，OSI模型在各层都进行差错校验和处理，而TCPIP仅在TCP层，即端到端进行差错控制，不可靠。
      • 都是分层协议，分层的功能基本相同
      • 层次数目不同，OSI中的网络层可以提供面向连接和无连接两种服务，而TCPIP中的IP层仅有无连接服务。
      • 一个是国际标准，一个是事实上的工业标准

    # 应用层

    • 把应用层协议数据单元可靠地从一个端系统传输到另一个端系统

    • 应用程序通过传输层的端口号字段的值区分不同的应用，在具体实现时时通过套接字（Socket）在网络上发送和接收应用报文，套接字由运输层逻辑地址（端口号）和网络层逻辑地址（IP地址）组成。

    • 应用层需要传输层协议提供的服务：

      • TCP协议：HTTP、SMTP、FTP、Telnet
      • UDP协议：SNMP
      • 都可以：DNS

    • 域名系统DNS

      • IP地址：32位二进制位标识
      • 主机名称解析
      • DNS是一个分布式数据库系统
      • 无连接的运输层协议UDP，53
      • 过程：
        • 主机提出域名解析请求，发送给本地域名服务器
        • 本地域名服务器查询本地域名缓存，有则返回，无则吧请求发给根域名服务器
        • 根域名服务器返回给本地域名服务器以各所查询域的主域名服务器的地址
        • 本地服务器再向上一步返回的域名服务器发送请求，进行迭代查询知道找到
        • 本地域名服务器把返回的结果保存到域名缓存，同时返回给客户机
      • DNS记录格式：Name, Value, Type, TTL（时间 ）

    • URL

      • 统一资源定位符，统一格式的Internet信息资源地址的标识方法
      • <协议>://<主机>:<端口>/<路径>

    • HTTP

      • 超文本传输协议，面向事务的应用层协议，无状态
      • 非持续性连接，每次浏览器和服务器交互都需要有两倍的RTT
      • 持续性连接，保持由TCP运输层协议建立起来的连接
      • 非流水线方式，客户机收到前一个报文后才能发出下一个请求报文
      • 流水线方式，客户机和服务器都可以连续发送请求报文和相应报文

    • FTP文件传输协议

      • 面向连接
      • 客户机和服务器之间建立并行的控制连接21和数据连接20
      • 控制连接是持久的，整个FTP回话期间一直保持
      • 数据连接是临时的，一次数据传输结束之后就拆除

    • 电子邮件

      • 系统：用户代理、电子邮件协议、电子邮件服务器

      • 电子邮件协议

        • 规定如何在两个互相通信的邮件进程之间交换信息

        • 用户代理使用SMTP协议发送邮件给发送发邮件服务器，发送方邮件服务器通过SMTP传输邮件给接收方邮件服务器，接收方邮件服务器通过POP3协议发送给用户代理

        • 简单邮件传输协议SMTP

          只能传送7位ASCII文本邮件 采用TCP连接 25

          工作过程

          • 用户代理建立与本地邮件服务器的连接，发送邮件给本地邮件服务器缓存下来
          • 本地邮件服务器每隔一定时间对缓存扫描，若有邮件发送，则建立与远地服务器的连接
          • 连接建立：SMTP客户机使用端口号25与SMTP服务器建立TCP连接，服务器回答ready表示服务就绪，连接建立，客户机发送hello命令给服务器，附上发送方主机名字，服务器回答250OK，表示已经准备好接收，否则回答421，表示服务不可用
          • 连接释放：当邮件发送完后，SMTP客户机发送QUIT命令，SMTP服务器回答关闭，表示同意释放TCP连接

        • 邮件读取协议POP3

          采用TCP连接 110

        • 邮件读取协议IMAP

          复杂，使用较少

    # 第五章 运输层

    • 运输层把尽力交付的不可靠网络服务演变成支持网络应用的可靠网络服务

    • 网络层为主机之间提供逻辑通信，运输层为应用进程之间提供端到端的逻辑通信

    • 网络寻址

      IP地址作为计算机设备的网络连接表示，用端口号作为应用程序的连接标识，构成套接字，表示一个应用服务的连接

    • 数据通信过程：

      • 连接建立

        三次握手

      • 数据传输

      • 连接释放

        突然释放、文雅释放（四次握手）

    • 运输协议实现的控制

      • 连接控制与缓冲管理
      • 差错控制，自动请求重发ARQ、校验
      • 序号设置
      • 超时机制，计时器

    • 实现可靠数据传输的网络协议设计

      • 滑动窗口机制

      • 停——等协议

        在发送方每发送完一个FDU后，就停止发送，等待接收方的确认数据，并把所发送过的数据的副本保留下来。若发送方收到的确认数据是ACK，则发送方按顺序接着发送下一个PDU；若收到的确认数据是NAK，则发送方就重新发送上次发送过的PDU

        • 差错控制 自动重发ARQ技术 反馈机制
        • 流量控制 由接收方控制发送方发送数据的速率
        • 重复数据处理 序号设置
        • 数据丢失处理 传输数据丢失或反馈数据丢失 设置超时计时器机制

      • 回退N协议

        发送方一次发送多个PDU，提高网络的传输效率

        • 累计确认机制
        • 捎带确认机制

      • 选择重传协议

        若发生差错，只重新传输出现差错涉及序号的PDU，减少不必要的重传

    • 拥塞控制

      • 因素

        网络节点处理器速度太慢、通信线路容量限制、节点缓冲区过小

      • 流量控制 局部控制

        数据链路层的相邻节点之间的流量控制、运输层的端节点之间的流量控制

      • 拥塞控制 全局控制

        对整个通信子网的流量进行控制，设计数据链路层、网络层、运输层

      • 开环控制

        通过良好设计，事先将有关发生拥塞的因素考虑周到，避免拥塞问题的出现

      • 闭环控制

        通过反馈机制，在工作过程中动态控制拥塞

    • UDP数据用户报协议

      • 不需要建立连接，剑圣连接建立所需要的时间
      • 协议简单，首部仅有8个字节，不需要维持包含许多参数、复杂状态表，尽力交付
        • 8个字节，4个字段，首部包括源端口、目的端口号、长度、校验和
      • 不支持拥塞控制，当出现拥塞时丢包处理
      • 面向报文
      • 支持一对一、一对多、多对一、多对多的交互通信

    • TCP传输控制协议

      • 面向连接的传输层协议

      • 提供双向同时通信和可靠的数据传输

      • 首部有20个字节，包括源端口和目的端口号、序号、确认好、数据偏移、标志位、接收窗口、校验和、选项

      • 流量控制

        通过报文段中的接收窗口字段，采用来自接收方的流量控制

      • 拥塞控制

        • 原因：PCU丢失增多，网络中传输时间增大
        • 对拥塞窗口的值进行动态的调控，采用慢速启动、快速增长、拥塞避免的机制，即由小到大逐渐增大发送方拥塞窗口的值，使向网络中发送PDU的速率更加合理
        • 加法增、乘法减

    # 网络层

    • 网络层的协议数据单元是数据包，网络层提供从源节点到目的节点之间的 可靠传输和分组的路由选择
    • 需要解决的问题：网络连接的建立与释放、路由选择、拥塞控制、网络互联、分片和重组、差错的检测与处理、服务选择、网络寻址
    • 虚电路服务
      • 面向连接
      • 用节点中的缓冲区和虚电路号标识一条逻辑信道连接
      • 避免所传输的分组在每一个节点都需要进行路由选择，途径的节点通过分组携带的虚电路号，查找虚电路转发表
    • 数据报服务
      • 无连接的
      • 不保证丢失和传输时延，“尽力交付”的服务
    • 路由选择
      • 路径选择
        • 通过路由选择算法计算出路由表，路由协议依据路由表的内容进行路径选择
      • 转发（交换）
        • PDU的目的网络地址不变，目的物理地址需要依次变成下一跳的物理地址
    • 静态路由选择
      • 人工完成，非自适应性路由选择，简单、开销小，用于小型的局域网
      • 固定路由选择算法
      • 随机路由选择算法
      • 扩散路由选择算法
    • 动态路由选择
      • 自适应性的路由选择，能较好地适应网络状态的变化，动态的改变和设置路由
      • 孤立式自适应路由选择
        • 热土豆算法
        • 反向探知算法
      • 距离矢量路由选择
        • Bellman-Ford算法
      • 链路状态路由选择
        • 链路状态算法
        • Dijkstra算法
      • 层次路由选择
        • 划分为多个自治系统
          • 具有一个单一和明确定义的路由选择策略
          • 内部路由协议、外部路由（网关）协议
    • IP协议
      • 尽力交付，无连接
      • 首部+数据，首部固定部分为20字节，包含12个字段，可选部分最长40字节
      • IP数据包的分片计算
      • IP地址：网络号+主机标识
      • IP地址实际标识的是网络节点的一个网络连接，不是网络节点（主机）的地址
    • 子网划分技术
      • 三级编址，网络标识+子网标识+主机号
      • 子网掩码
    • 无分类编址（CIDR）
      • 在变长子网掩码基础上发展起来，可以更有效地分配IP地址空间
      • 两级编址， 网络前缀+主机标识
    • 控制报文协议ICMP
      • 用于监视与检测网络、报告意外事件的发生
      • 不能保证可靠传输，ICMP报文可以会丢失
      • 分片后的IP分组，只有偏移为0的第一个分片IP分组才能使用ICMP
      • ICMP差错报告报文：目的不可达；源端节点抑制；路由重定向；时间超时；参数问题；查询报文：回送请求或应答；路由器通告或询问；时间戳请求或应答
    • 路由选择协议
      • 内部路由协议1——RIP路由信息协议
        • 基于Bellman-Foed算法的距离矢量路由协议
        • 路由器周期地与相邻路由器交换路由信息报文
        • 根据最短路径算法更新各自的路由表，通过计时器维护路由表的完整性
      • 内部路由协议2——OSPF开放最短路径优先协议
        • 采用链路状态路由选择算法
        • Dijkstra最短路径算法
      • 外部路由协议——BGP4
        • 路径矢量路由选择协议
        • PV算法
    • 网络互联设备
      • 中继器
      • 网桥
      • 路由器
      • 协议转换器

    # 第七章 数据链路层和局域网

    • 提供网络中相邻节点之间的可靠数据传输，把原始的、由差错的物理传输线路加上数据链路协议之后，构成逻辑上可靠的数据链路
    • 两种信道：点对点信道：广域网；广播信道：局域网
    • 信道访问协议
      • 为了提高通信资源的利用率，广泛采用多个通信实体共同共享一条公共信道的方法
      • 可以通过信道复用技术、多路访问协议实现
      • 固定信道划分协议
        • 时分复用TDM
        • 频分复用FDM
        • 码分多址CDMA
        • 统计时分复用STDM
      • 随机访问协议
        • 适合广播信道
        • 纯ALOHA
        • 时隙ALOHA
        • CSMA载波侦听多路访问协议
        • CSMA/CD载波侦听多路访问/冲突检测协议
      • 轮流访问协议
    • MAC地址与地址解析协议ARP
      • MAC地址
        • 48位二进制位标识，固化在网卡的ROM芯片中，全球唯一
      • ARP
        • 通过ARP进行IP地址到MAC地址的转换
